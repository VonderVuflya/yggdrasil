# Yggdrasil MVP AI Memory Index

Purpose: give MCP-compatible agents one small boot map for durable memory, large-output handling, and human-readable materialization.

## Layers

- Durable memory backend: Muninn REST/MCP. Use it for project decisions, debugging lessons, failed approaches, workflows, and reusable instructions.
- Large-output/session layer: context-mode. Use it for logs, huge command output, repository scans, and recovery after compaction.
- Human-readable view: Obsidian vault materialized from Muninn records. Do not treat Markdown as the runtime retrieval engine.

## Before Non-Trivial Work

1. Identify `project`.
2. Search/bootstrap Muninn for relevant `debugging_lesson`, `decision`, `failed_approach`, `workflow`, and project instructions. In this demo, use `scripts/ygg.py bootstrap --project PROJECT --query "..."`
3. Use context-mode for large files/output instead of loading raw bytes into the prompt.
4. If memory conflicts with the repository, the repository wins and the memory should be updated or marked stale.

## After Meaningful Work

Save reusable memory when the work produced a lesson that should survive the session.
In this demo, use `scripts/ygg.py remember --project PROJECT --type TYPE --json-file lesson.json`.

Preferred types:
- `debugging_lesson`
- `decision`
- `failed_approach`
- `workflow`
- `docs_drift`
- `skill_candidate`

Debugging lessons must capture:
- problem
- symptoms
- failed approaches
- working solution
- recognition signals
- commands/files/evidence
- next_time guidance

## Safety

- Never save secrets, credentials, private tokens, `.env` values, customer data, or raw sensitive logs.
- Project isolation is mandatory: project memories use `scope="project"` and `metadata.project`.
- Global memories are only for stable cross-project preferences and universal rules.
- Use `MUNINN_PROJECT_SCOPE_STRICT=1` for local agent workflows.
