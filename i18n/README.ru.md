С каждым новым чатом ваш ИИ всё забывает. Вы заново объясняете проект, решения, подводные камни — каждый раз, в каждом инструменте. **Yggdrasil — это крошечный, всегда работающий «мозг памяти», к которому подключается любой агент.** Откройте новую сессию в любом проекте, с любым ИИ — и он уже знает, что вы решили, что сломалось и что осталось незакрытым, а в фоне он тихо учится на вашей работе.

```text
$ cd ~/projects/checkout-api && claude        # a brand-new session

🌳 Yggdrasil  (injected automatically at session start)
   You are Yggdrasil — your persistent assistant across tools and projects.
   Open follow-ups & status:
   • [project_status] payments refactor: idempotency keys added; open: e2e tests
   Durable memory for `checkout-api`:
   • [debugging_lesson] webhook 401 → signing secret rotated; update env + redeploy

> "have I solved a flaky websocket reconnect anywhere before?"

🌳 recall → found in project `realtime-dash`:
   refresh the token *before* opening the socket, then retry with capped backoff.
```

Никаких «давай я напомню, что мы делали вчера». Всё просто на месте.

## ❌ Без Yggdrasil

- 🔁 Вы заново объясняете контекст проекта ИИ в каждом новом чате.
- 🧩 Уроки, извлечённые в одном проекте, никогда не доходят до следующего.
- 🤖 Перешли с Claude Code на Codex → новый инструмент не знает ничего.
- 🗑️ Выстраданные открытия при отладке исчезают, как только сессия заканчивается.

## ✅ С Yggdrasil

- 🧠 **Долговечная память** — решения, уроки и статус сохраняются между сессиями.
- 🔌 **Любой агент, один мозг** — Claude Code, Codex и любой MCP-хост используют общую память.
- 🌐 **Межпроектный поиск** — *«похоже на то, что ты делал в проекте B — переиспользовать?»*
- 🌱 **Самообучение** — локальная модель консолидирует память в фоне (ноль API-токенов).
- 🪪 **Душа** — дайте ей имя и характер; она проявляется одинаково в каждом инструменте.
- 🔒 **100% локально и приватно** — ваша память живёт на вашей машине. Никакого облака, никаких аккаунтов.

## 🚀 Быстрый старт

> **Требования:** macOS, Python 3.10+. Опционально (для семантического поиска): [Ollama](https://ollama.com).

```bash
git clone https://github.com/your-org/yggdrasil.git
cd yggdrasil
scripts/install.sh install          # interactive wizard — detects your hardware,
                                    # recommends models, sets up the background service
```

Вот и всё. Установщик:
1. 🔍 определяет ваши CPU/RAM/GPU и **рекомендует модели, подходящие вашей машине**,
2. 🔑 генерирует приватный токен аутентификации (никогда не зашит в код),
3. 🛎️ устанавливает всегда работающую фоновую службу (автозапуск при входе, перезапуск при сбое),
4. 🤝 регистрирует инструменты памяти в **Claude Code и Codex**,
5. 🪝 (опционально) включает хук на старте сессии, который автоматически подставляет память вашего проекта.

Хотите просто попробовать движок без установки службы?

```bash
python3 scripts/ygg_memory_server.py --reset --db /tmp/ygg.sqlite   # runs on :42069
scripts/run_gates.sh                                                # see all checks pass
```

## 🧠 Как это работает

Yggdrasil — это **память + инструменты**; *интеллект* — это ваша LLM. Он лишь следит за тем, чтобы нужная память оказалась перед нужным агентом в нужный момент.

```text
   Claude Code / Codex / any MCP host
                 │   (MCP tools: ygg_search, ygg_recall, ygg_remember … )
                 ▼
        ┌─────────────────────┐      SessionStart hook injects
        │  Yggdrasil engine    │◀──── identity + project memory + open follow-ups
        │  (always-on daemon)  │
        │  SQLite + FTS5       │      background local model (optional)
        │  + optional vectors  │────▶ dedupes / links / consolidates memory
        └─────────────────────┘
                 │ materializes to
                 ▼
          📓 Obsidian vault (human-readable, editable)
```

- **Движок** — HTTP-сервер на одной только стандартной библиотеке поверх SQLite + FTS5. Ноль зависимостей, ~21 МБ RAM.
- **Извлечение** — лексическое по умолчанию; добавьте локальную embedding-модель для семантического и кросс-язычного поиска.
- **Управление** — дубликаты / устаревшие / противоречащие воспоминания выносятся на ревью; изменения неразрушающие (архивирование, а не удаление).
- **Obsidian** — каждое воспоминание — это ещё и Markdown-заметка, которую можно читать и редактировать.

## ⭐ Возможности

- 🧠 **Долговечная межсессионная память** для любого MCP-совместимого агента.
- 🌐 **Межпроектный поиск** + проактивный контракт «ты уже решал это раньше».
- 🔎 **Гибридное извлечение** — BM25 + опциональные плотные эмбеддинги, объединённые вместе; кросс-язычное (например, EN↔RU).
- 🪪 **Личность / персона** подставляется в каждой сессии (ощущение «Джарвиса»).
- 📌 **Статус и незакрытые задачи** показываются на старте сессии — *«какой статус у X?»* отвечается мгновенно.
- 🌱 **Фоновое самообучение** — небольшая локальная модель консолидирует память (по умолчанию только предлагает изменения).
- 🧹 **Цикл управления** — очередь на ревью + неразрушающее архивирование/слияние.
- 📓 **Материализация в Obsidian** — читаемо, редактируемо, переносимо.
- 🔒 **Локально и приватно** — никакого облака, никаких аккаунтов, ваши данные остаются на месте.
- 🪶 **Ноль жёстких зависимостей** — чистая стандартная библиотека Python; опциональные локальные модели через Ollama.

## 🛠️ Команды

**CLI — `scripts/ygg.py`** (операции с памятью со стороны агента)

| Команда | Что делает |
| --- | --- |
| `health` | Проверить, что движок жив |
| `bootstrap --project P` | Подтянуть память проекта перед началом работы |
| `search --project P --query "…"` | Поиск в рамках проекта (`--type`, `--limit`, `--json`) |
| `recall --query "…"` | **Межпроектный** поиск — «я где-нибудь это уже делал?» |
| `remember --project P --type debugging_lesson --content "…"` | Сохранить долговечное воспоминание (с защитой секретов, дедупликацией) |
| `materialize --id ID --project P` | Экспортировать одно воспоминание в заметку Obsidian |

**Служба — `scripts/install.sh`** (жизненный цикл и настройка)

| Команда | Что делает |
| --- | --- |
| `install` | Пошаговый мастер → фоновая служба + регистрация MCP |
| `recommend` | Показать каталог моделей с учётом вашего железа |
| `status` · `start` · `stop` · `restart` · `logs` | Управление всегда работающим демоном |
| `hooks` · `unhooks` | Включить/выключить хук авто-bootstrap на старте сессии |
| `consolidate` · `unconsolidate` | Запланировать/удалить фоновую консолидацию памяти |
| `token` · `uninstall` | Вывести токен аутентификации · удалить службу + регистрацию |

**MCP-инструменты** (то, что видят агенты): `ygg_health`, `ygg_bootstrap`, `ygg_search`, `ygg_recall`, `ygg_remember`, `ygg_materialize`.

## 🔌 Используйте вместе со своим агентом

- **Claude Code** — после `install.sh install` инструменты зарегистрированы (`/mcp` показывает `yggdrasil`), а хук на старте сессии автоматически подставляет память. Просто откройте проект и работайте.
- **Codex** — тоже зарегистрирован; подтвердите вызов инструмента `ygg_*` один раз за сессию.
- **Любой MCP-хост** — направьте его на `scripts/ygg_mcp_server.py` (stdio) с `YGG_MUNINN_URL` + `YGG_MUNINN_TOKEN`.

Задайте ей характер — отредактируйте `~/.yggdrasil/identity.json`:

```json
{ "name": "Jarvis", "persona": "concise, proactive, dry wit", "user_facts": ["prefers TypeScript", "ships small PRs"] }
```

## 📊 Потребление ресурсов и качество

**Потребление ресурсов** (измерено, 13 воспоминаний): **~21 МБ RAM**, **~0% CPU в простое**, **ноль зависимостей** (стандартная библиотека Python 3.10+). Диск ≈ десятки КБ на воспоминание. Плотный поиск опционален и добавляет локальную модель Ollama (например, `all-minilm`, 45 МБ).

**Качество извлечения** (`eval/ygg_eval.py`, recall@1):

| Режим | recall@1 | перефразирование | кросс-язычный (EN→RU) |
| --- | --- | --- | --- |
| лексический (по умолчанию) | 0.63 | 0.00 | 0.00 |
| плотный · `all-minilm` (45 МБ, EN) | 0.75 | 0.67 | 0.00 |
| плотный · `paraphrase-multilingual` (~560 МБ) | **0.94** | 0.67 | **1.00** |

Запросы `keyword` и `identifier` дают 1.0 в любом режиме. Проверьте сами: `python3 eval/ygg_eval.py`.

## ❓ Частые вопросы

<details>
<summary><b>Отправляет ли он мой код или память в облако?</b></summary>

Нет. Движок, база данных и опциональные модели — всё работает локально. Никаких аккаунтов, никакой телеметрии. Ваша память никогда не покидает вашу машину.
</details>

<details>
<summary><b>Чем это отличается от Context7 / RAG по документации?</b></summary>

Context7 подтягивает актуальную <i>публичную документацию библиотек</i> (свежий API React/Next.js). Yggdrasil помнит <i>вашу собственную работу</i> — ваши решения, уроки и статус проекта. Они дополняют друг друга; используйте оба. См. <a href="#-yggdrasil-в-сравнении-с-альтернативами">сравнение</a>.
</details>

<details>
<summary><b>Запоминает ли он всё автоматически?</b></summary>

Нет — и это намеренно. Извлечение происходит автоматически; <i>запись</i> же осознанна (агент вызывает <code>ygg_remember</code> для долговечных уроков). Автозахват всего загрязняет память, поэтому мы так не делаем. Фоновая модель консолидирует уже сохранённое (по умолчанию только предлагает).
</details>

<details>
<summary><b>Нужен ли мне GPU или API-ключ?</b></summary>

Нет. По умолчанию используется чисто лексический поиск — ноль зависимостей, мгновенно. Семантический поиск подключается по желанию и использует <i>локальную</i> модель через Ollama (без API-ключа). Установщик порекомендует модель, подходящую вашему железу.
</details>

<details>
<summary><b>Сколько токенов это стоит моему агенту?</b></summary>

Очень мало. Старт сессии подставляет ~300 токенов памяти; каждый вызов инструмента возвращает небольшой фрагмент. Вся тяжёлая работа (индексация, эмбеддинги, консолидация) выполняется вне LLM на вашей машине.
</details>

<details>
<summary><b>Могу ли я редактировать или удалять воспоминания вручную?</b></summary>

Да. Воспоминания материализуются в Markdown-заметки в хранилище Obsidian — читайте, редактируйте или удаляйте их как любой файл. Движок никогда не удаляет безвозвратно; он архивирует (обратимо).
</details>

<details>
<summary><b>Готов ли он к продакшену?</b></summary>

Это честная <b>альфа</b>: основной сценарий и полный цикл управления покрыты проходящими проверками (<code>scripts/run_gates.sh</code>). Пока не закалён для многопользовательского/продакшен-использования.
</details>

## 🆚 Yggdrasil в сравнении с альтернативами

| | **Yggdrasil** | Context7 | Обычная память LLM |
| --- | --- | --- | --- |
| Знает **ваши** решения/уроки | ✅ | ❌ | ⚠️ в пределах одной сессии |
| Актуальная публичная документация библиотек | ❌ (используйте Context7) | ✅ | ❌ |
| Между сессиями и между **агентами** | ✅ | ✅ | ❌ |
| Межпроектный поиск | ✅ | — | ❌ |
| Записывает/накапливает вашу память | ✅ | ❌ (только чтение) | ⚠️ |
| Локально и приватно | ✅ | ☁️ в облаке | зависит |
| Самоконсолидирующийся | ✅ | ❌ | ❌ |

**Короче:** Context7 = корректный API *чужой* библиотеки. Yggdrasil = память о *вашей собственной* работе. Используйте оба.
